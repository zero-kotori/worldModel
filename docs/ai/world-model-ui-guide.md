# World Model UI 使用说明

本文说明 `/admin/world-model/*` 后台界面的实际功能和推荐使用顺序。系统默认面向管理员个人使用，不做公开预测、交易执行或投资建议。

## 1. 推荐操作顺序

最稳妥的使用路径是：

1. 在 `/admin/world-model/beliefs` 创建信念表和至少两个假设。
2. 在 `/admin/world-model/sources` 添加或补齐来源配置。
3. 先运行来源 dry-run 或“预检闭环”，确认能采集到真实观察。
4. 在 `/admin/world-model/observations` 处理待审候选、未知观察和重复候选。
5. 在 `/admin/world-model/evidence` 将观察确认为证据，检查证据-假设关联和概率更新预览。
6. 确认无误后应用更新；需要撤销时在“更新回滚”里回滚。
7. 稳定后再在 `/admin/world-model/sources#automation-worker` 启动 worker。

## 2. 总览页

路径：`/admin/world-model`

总览页用于看系统当前是否健康：

- 信念、活跃假设、待处理观察、已确认信据、最近更新等摘要。
- 自动化健康状态：最近采集、最近候选、自动应用数、待审数、连续失败数、worker 状态。
- 下一步操作入口：检查来源、处理待审、启动守护进程、查看低影响样本、查看模型配置等。
- 全局图谱和最近更新记录。

如果总览页提示自动闭环有风险，优先处理它给出的下一步入口，而不是直接开启自动应用。

## 3. 信念页

路径：`/admin/world-model/beliefs`

信念页管理“要判断的问题”和候选假设。

主要功能：

- 新建信念表：填写标题、分类、概率结构、描述和初始假设。
- 查看每个假设的当前概率、状态、证据影响、时间窗口。
- 给已有信念添加新假设。
- 查看推荐假设，并一键添加。
- 从未知观察跳转过来时，可以基于该观察生成新的信念和假设。
- 页面内嵌图谱，便于查看信念、假设、观察、证据和更新之间的关系。

概率结构含义：

- `MUTUALLY_EXCLUSIVE`：同一信念下的假设互斥，概率会归一化，合计为 1。
- `INDEPENDENT`：假设彼此独立，每个假设单独更新，合计不要求为 1。

## 4. 来源页

路径：`/admin/world-model/sources`

来源页管理外部采集、自动闭环和 worker。

### 推荐来源

“推荐来源”区域列出项目内置 preset：

- RSS：Hacker News、arXiv、Hugging Face Blog、Google News Query RSS 等。
- GitHub Repository Query。
- Hugging Face Model Query。
- GDELT Document Query。
- Polymarket Query 和 Polymarket Events Query。
- Reddit Public Query。
- X Recent Search。

未添加的 preset 只显示“添加”，不会参与自动闭环。点击“补齐推荐来源”会一次性创建缺失 preset。

当前限制：自动闭环的“限定来源”和 worker 的“Worker 来源”只列出已经创建的来源配置。未添加的 preset 不会出现在多选框里，也没有 `!` 标记。后续如果要显示未添加 preset，可以用 `!` 标记并默认不选中。

### 来源配置

新增或编辑来源时主要字段：

- 名称：后台显示名。
- 类型：`RSS`、`WEB_PAGE`、`SEARCH`、`GITHUB`、`HUGGING_FACE`、`GDELT`、`PREDICTION_MARKET`、`SOCIAL`。
- URL：采集地址；带 `{query}` 的 URL 会按假设查询词展开。
- Adapter：解析器名称，例如 `rss`、`polymarket_markets`、`polymarket_events`、`x_recent_search`。
- 凭据引用名：只保存引用名，不保存真实 key/token。真实值放环境变量。
- 可信度：来源整体可信度，会折扣证据的似然影响。
- 自动确认阈值：自动应用证据时的最低关联置信阈值。
- 启用：未启用来源不会被自动闭环选择。
- 自动确认：允许该来源在满足阈值和安全策略时自动确认。

### 运行来源

“运行来源”区域有三种入口：

- 采集入库：真实抓取并写入观察池。
- 仅采集待审：写入观察，但不自动确认。
- Dry-run：使用表单里的样本标题、链接、正文跑一遍解析和处理，不抓真实远端。

### 自动证据闭环

自动闭环会执行：

1. 从活跃假设生成查询词。
2. 选择启用来源。
3. 调用来源 adapter 采集观察。
4. 对观察去重。
5. 推荐证据-假设关联。
6. 根据阈值、安全策略和来源配置决定进入待审或自动应用。

表单字段：

- 限定信念：只为选中的信念生成查询；不选表示不限定。
- 限定来源：只跑选中的已配置来源；不选表示从所有启用、非 `MANUAL` 来源中选择。
- 单次最大查询：本轮最多生成多少查询词。
- 单次最大来源：本轮最多跑多少来源。
- 单次最大观察：每个来源最多处理多少观察。
- 候选识别阈值：观察和假设匹配的最低相关阈值。
- 自动应用阈值：证据关联可自动应用的最低置信阈值。
- 重复候选：本轮遇到重复观察后的处理方式，默认拒绝。
- 未知观察：本轮遇到未匹配观察后的处理方式，默认保留。
- 低影响观察：本轮遇到低相关性观察后的处理方式，默认保留。
- 仅生成待审：勾选后不会自动确认或自动应用。
- 补齐推荐来源：未限定来源时，运行前自动创建缺失 preset。
- 本次自动应用：本轮允许自动确认；仍会经过 LLM 评估、来源质量和覆盖保护。

来源为空时不是“不跑来源”，而是“不限定来源”。系统会从已启用、非 `MANUAL` 来源里按稳定性排序选择，并受“单次最大来源”限制。

清理策略含义：

- `保留`：观察继续留在对应队列，等待人工处理。
- `拒绝`：观察状态改为 `REJECTED`，从待处理队列中移出，保留审计记录。
- `删除`：观察状态改为 `DELETED`，从队列和图谱中隐藏，仍然是软删除，不物理删除数据库行。

### Worker

worker 是无人值守的自动闭环定时器。

UI 中“启动守护进程”会保存 worker 配置并启动进程内 worker：

- Worker：worker id，默认 `default`。
- Worker 信念：限定要观察的信念。
- Worker 来源：限定要跑的已配置来源；为空表示不限定。
- 间隔秒：两轮运行的基础间隔。
- 最大查询、最大来源、最大观察：单轮工作量上限。
- 候选阈值、应用阈值：同自动闭环。
- 重复候选、未知观察、低影响观察：同自动闭环的清理策略，会被保存到 worker 配置并在每轮运行时使用。
- 失败退避：连续失败时把下次运行间隔放大。
- 最长间隔秒：退避后的最大等待时间。
- 仅生成待审：保持人工审核优先。
- 补齐推荐来源：未限定来源时自动创建缺失 preset。
- 自动应用：允许满足安全策略的候选自动确认并应用。

生产 Linux 服务器推荐用 `npm run observe:worker` 或 systemd/cron/supervisor 拉起外部 worker。UI 里的进程内 worker 适合本地和单进程部署，但进程重启时依赖启动恢复逻辑。

## 5. 观察页

路径：`/admin/world-model/observations`

观察是“采集到但还未确认成证据”的原始材料。

主要区域：

- 手动观察：人工录入标题、链接、作者、可信度、正文、hash 和语义 key。
- 待审候选：系统认为可能能转成证据的观察，可确认推荐、调整关联、结算假设或拒绝。
- 待处理观察：普通待处理观察，可转为证据或拒绝。
- 未知证据队列：未匹配、低影响或需要人工判断的观察，可批量拒绝或批量软删除。
- 重复候选：系统认为重复的观察，可批量拒绝或批量软删除。
- 观察池：全部观察列表，支持基础字段编辑。

常用处理方式：

- 推荐关联看起来合理：点“确认推荐”。
- 推荐关联需要调整：点“调整”，跳到证据页修改方向、相关性、似然比和置信度。
- 观察有价值但没有对应假设：点“推荐假设”或“补假设”。
- 明显重复或低质量：拒绝；如果确认只是噪声并希望从队列和图谱中隐藏，可以删除。删除是软删除状态，不会物理抹掉审计记录。

## 6. 证据页

路径：`/admin/world-model/evidence`

证据是已经确认会参与概率更新的观察。

主要功能：

- 运行自动搜证闭环。
- 从观察确认为证据。
- 手动录入证据并直接关联假设。
- 查看证据影响图谱。
- 查看证据库、证据-假设关联、更新预览。
- 应用更新。
- 编辑证据和关联，保存后重算。
- 拒绝或删除未应用证据。
- 回滚已应用的更新事件。

证据关联字段：

- 方向：支持、反对、混合、中性。
- 相关性：证据和假设是否相关。
- 似然比：大于 1 表示支持假设，小于 1 表示削弱假设，接近 1 表示影响很小。
- 置信度：这条关联本身的可靠程度。
- 解释：人工或模型给出的理由。

“应用更新”会把当前概率变成 posterior，并写入一个可审计的更新事件。回滚会用该事件保存的 prior snapshot 恢复概率。

## 7. 图谱页

路径：`/admin/world-model/graph`

图谱页用于查看和编辑关系：

- 节点：来源、信念、假设、观察、证据、更新事件。
- 边：来源产生观察，观察确认成证据，证据关联假设，证据产生更新事件。
- 支持通过 URL 参数聚焦某个信念、假设、来源、证据或更新。
- 点击节点后可以切换“只看相关 / 显示全部”，用于只保留与当前信念、假设、观察或证据直接相关的子图。
- 右侧编辑区会随选中节点或边变化，支持编辑来源、信念、假设、观察、证据和证据关联。
- 归档的信念/假设、已拒绝或已删除的观察、已删除证据不会出现在默认图谱中。
- 选中待处理观察时，可以在右侧工作区拒绝观察；拒绝后的观察不应再确认为证据。

图谱适合做复盘：某个概率为什么变了、由哪条证据触发、是否需要回滚。

## 8. 模型页

路径：`/admin/world-model/models`

模型页管理 LLM、训练数据、模型产物和似然评分审计。

主要区域：

- LLM API 配置：显示主评分器和假设推荐器是否已配置。
- LLM 评估结果：显示最近一次真实样本评估和自动应用保护状态。
- LLM 评估运行：生成评估结果，用于判断自动应用是否安全。
- 真实训练数据抓取：抓取公开真实样本，写入 `model-artifacts/`。
- 轻量模型训练：准备样本、训练并导入轻量模型。
- 模型产物导入：导入 `LIGHTWEIGHT`、`LLM`、`DEEP_ADAPTER` 产物。
- 模型状态：列出现有模型产物。
- 似然评分审计：列出最近 likelihood run，包括证据、假设、似然比、置信度、模型版本和解释。

DeepSeek 默认配置使用：

```env
LLM_PROVIDER="deepseek"
LLM_BASE_URL="https://api.deepseek.com"
LLM_API_KEY=""
LLM_MODEL="deepseek-chat"
```

外部 deep-model 是独立配置，不会自动复用 `LLM_*`：

```env
EXTERNAL_MODEL_ENDPOINT=""
EXTERNAL_MODEL_API_KEY=""
EXTERNAL_MODEL_MODEL=""
EXTERNAL_MODEL_VERSION=""
EXTERNAL_MODEL_TIMEOUT_MS="30000"
```

## 9. 安全默认值

- 来源 preset 的 `autoConfirm` 默认关闭。
- 自动闭环表单默认勾选“仅生成待审”。
- worker 是否自动应用由 worker 配置控制；开启前应先跑 dry-run 和预检。
- X/Twitter 需要 `X_MAIN_BEARER_TOKEN`，Polymarket 公共查询当前不需要 API key。
- 凭据只放环境变量或本地密钥，不写入数据库和 Git。
- 回滚只能回滚更新事件，不会删除原始观察或证据。
